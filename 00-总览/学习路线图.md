# AI 学习路线图

## 第一阶段：建立整体地图

目标：先知道 AI 世界里分别有什么，而不是一开始钻细节。

重点：

1. 机器学习、深度学习、大模型分别是什么
2. Transformer 为什么重要
3. Prompt、RAG、Fine-tuning、Agent 分别解决什么问题
4. 一个 AI 应用从模型到产品的基本链路

完成标志：

- 你能用自己的话解释核心概念
- 你能区分常见名词，不再混淆

## 第二阶段：补齐关键原理

目标：建立足够扎实的技术判断力。

重点：

1. Token、Embedding、Attention
2. 训练、微调、推理
3. 上下文窗口、延迟、成本、效果的关系
4. 常见评估指标和幻觉问题

完成标志：

- 你能判断一个方案“为什么有效”
- 你能识别常见营销话术和概念包装

## 第三阶段：掌握工程实现

目标：从“知道概念”进入“知道怎么做”。

重点：

1. Prompt 设计与结构化输出
2. RAG 基本链路与检索质量
3. Agent 工作流、工具调用、记忆机制
4. 评测、日志、缓存、部署

完成标志：

- 你能看懂大部分 AI 应用架构图
- 你能自己搭一个简单原型

## 第四阶段：形成应用判断

目标：把知识转换成场景能力。

重点：

1. 哪些问题适合用 AI
2. 哪些问题不适合
3. 成本、准确率、可控性如何权衡
4. 如何从需求倒推技术方案

完成标志：

- 你能独立拆解一个 AI 产品场景
- 你能提出更合理的技术路线

## 第五阶段：沉淀自己的知识资产

目标：把学习成果转成长期可复用资产。

重点：

1. 概念卡片
2. 主题地图
3. 方案模板
4. 案例库
5. 复盘与输出

完成标志：

- 你有自己的 AI 知识网络
- 你能持续对外输出或内部分享
