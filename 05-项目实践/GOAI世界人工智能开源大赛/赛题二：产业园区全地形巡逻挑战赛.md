# 赛题二：产业园区全地形巡逻挑战赛

## 来源

- 来源类型：你提供的赛题说明图
- 赛题名称：产业园区全地形巡逻挑战赛

## 一句话总结

这道题要求参赛者基于指定四足机器人和官方 SDK，在仿真地图与真实环境中，做出一个能在动态未知环境里自主完成多巡检点任务的机器人系统，并提交可运行 Demo 和技术方案。

## 这道题到底在考什么

这道题表面上是“巡逻”，本质上考的是一个移动机器人系统能不能闭环跑通。

它不是只考：

- 识别
- 导航
- 避障

而是一起考下面几层能力：

1. 环境理解
2. 自主导航
3. 实时避障
4. 路径优化
5. 多目标任务调度
6. 仿真到真机迁移
7. 工程化集成与稳定运行

所以这更像一道“系统整合题”，不是单点模型题。

## 题目输入条件

图里给了 4 个基础条件。

### 1. 指定硬件平台

基于 `山猫四足机器人`。

这意味着参赛方案默认不能脱离官方机器人本体，必须围绕它的：

- 机身
- 四足
- 传感器
- 控制单元
- 电池
- 通信模块

来做系统集成。

## 2. 官方二开接口和 SDK

图里强调使用官方开放接口与开发工具包。

这说明比赛不只是允许“调参数”，而是明确鼓励二次开发，例如：

- 读取传感器数据
- 控制机器人运动
- 接入外部设备
- 调用 API
- 使用官方示例 Demo 做扩展

这类题的现实含义是：

- 重点不是从零造硬件
- 而是在既定平台上做出更聪明、更稳的系统能力

### 3. 仿真地图

比赛提供仿真测试环境。

价值在于：

- 可以先做算法开发
- 可以反复调试和验证
- 可以降低真机调试成本

这意味着一个合理的参赛策略通常是：

- 先在仿真里验证主流程
- 再逐步迁移到真实场地

### 4. 真实环境条件

比赛还有真实场景测试。

图中强调的真实要素包括：

- 实际地形
- 光照
- 障碍物
- 通道
- 巡检点位置

这意味着系统不能只在“理想地图”里跑通，还要能处理现实世界噪声。

## 题目的核心任务是什么

图里的比赛任务可以压缩成一句：

在动态未知环境下完成多目标巡检。

这句话里有 3 个关键词。

### 1. 动态

说明环境不是静止的，可能会出现：

- 行人移动
- 物体位置变化
- 临时障碍

### 2. 未知

说明机器人面对的信息不是完全提前给定的。

也就是说：

- 地图可能不完整
- 实际环境和仿真环境会有偏差
- 机器人需要边走边观察边修正

### 3. 多目标巡检

说明不是去一个点就结束，而是要：

- 到达多个巡检点
- 按任务要求完成检查、拍摄、识别、记录等动作
- 合理安排顺序和时间

所以它本质上是一个“任务链”，不是单次导航。

## 图里点名的 4 个核心能力

### 1. 自主导航

机器人要知道：

- 我在哪里
- 目标点在哪里
- 我该怎么走过去

这通常对应：

- 定位
- 建图或地图使用
- 全局导航

### 2. 实时避障

机器人要在行进中持续观察环境，并在发现障碍时及时绕开或停下。

这通常对应：

- 局部感知
- 障碍检测
- 局部避障
- 安全策略

### 3. 路径优化

不是“能到就行”，而是要在多条可行路径中选择更优路线。

图里强调的优化目标包括：

- 更短
- 更安全
- 更高效
- 更稳定

### 4. 任务调度

机器人需要决定：

- 多个巡检点先去哪个
- 后去哪个
- 如何安排顺序
- 如何保证整体任务完成率

这是这道题比较像“具身智能系统题”的地方，因为它不只是控制，还涉及任务层决策。

## 用通俗话解释图里的核心名词

### 动态未知环境

环境不是固定不变的，而且很多信息事先不知道。

例如：

- 人突然出现
- 箱子被移动
- 临时通道被堵住

### 自主导航

机器人自己知道目标在哪、自己规划路线、自己走过去。

### 实时避障

机器人一边走一边看，一旦发现障碍，立刻绕开或停下，保证安全。

### 路径优化

在多条路都能走的情况下，选更合理的路线，而不是盲目走最近直线。

### 任务调度

多个点都要巡检时，机器人自己安排先后顺序和执行节奏。

### 多目标巡检任务

机器人不只是“移动到点”，还要在多个点位上完成检查、拍摄、识别、记录等任务动作。

## 这道题交付什么

图里明确给了两个最终提交物。

### 1. 可运行 Demo

这不是 PPT，也不是算法说明，而是：

- 一个完整程序/系统
- 能在仿真或真机上运行
- 能跑通整个巡检流程

### 2. 技术方案

需要提交技术文档，通常要说明：

- 系统设计
- 算法原理
- 功能模块
- 实现方法
- 实验结果
- 创新点

## 从技术角度看，这道题最可能的系统结构

如果按工程系统拆分，一个较标准的方案通常会包含下面几层：

### 1. 感知层

负责读取和处理：

- 相机
- 激光雷达或深度信息
- IMU
- 里程计
- 机器人状态数据

### 2. 定位与地图层

负责：

- 机器人位姿估计
- 地图构建或地图匹配
- 巡检点定位

### 3. 规划与导航层

负责：

- 全局路径规划
- 局部路径调整
- 动态避障

### 4. 任务决策层

负责：

- 多点任务排序
- 优先级调整
- 巡检状态管理
- 异常重试或回退逻辑

### 5. 执行控制层

负责：

- 将导航与策略输出转成机器人运动控制
- 对接官方控制接口

### 6. 评测与日志层

负责：

- 记录轨迹
- 记录任务完成情况
- 记录避障事件
- 支持复盘和性能优化

## 这道题和具身智能的关系

这道题虽然不一定直接要求上 `VLA` 或 `WAM`，但它和具身智能的核心问题高度一致：

- 要在真实环境里行动
- 要感知和决策结合
- 要在不完全已知条件下完成任务
- 要根据环境变化持续调整

如果用更技术化的语言说，它考的是：

- `Perception`
- `Navigation`
- `Planning`
- `Control`
- `Task Orchestration`

而不是单纯的语言理解或图像识别。

## 这道题的难点在哪里

### 1. 仿真跑通不代表真机能跑通

真实环境里会多出很多问题：

- 光照变化
- 传感器噪声
- 地面摩擦差异
- 定位漂移
- 障碍物不规则移动

### 2. 多目标任务比单点导航复杂很多

因为它要同时考虑：

- 点位顺序
- 时间成本
- 局部异常
- 任务中断恢复

### 3. 避障和路径优化会互相影响

最短路径未必最安全，最安全路径未必最快。

### 4. 系统集成比单模块效果更重要

比赛最后看的是：

- 能不能稳定跑完整流程

而不是某一个子算法单项指标有多高。

这不是单点导航题，而是一道围绕四足机器人平台展开的系统集成题。比赛重点是把感知、定位、规划、避障、调度和真机控制串成一个完整闭环，并在仿真和真实环境中稳定运行。
